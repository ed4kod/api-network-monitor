# Мониторинг сетевых устройств

Веб-приложение для мониторинга доступности сетевых устройств с помощью ICMP-запросов (ping). Позволяет отслеживать состояние устройств в реальном времени и получать уведомления об изменении их статуса.

## Возможности

- Добавление, редактирование и удаление устройств для мониторинга
- Отслеживание доступности устройств в реальном времени
- Просмотр истории изменения статуса устройств
- Запуск и остановка мониторинга для отдельных устройств или всех сразу
- Веб-интерфейс для удобного управления

## Требования

- Python 3.7+
- FastAPI
- Uvicorn
- Jinja2
- Python-multipart

## Установка

1. Клонируйте репозиторий:
```
git clone https://github.com/ed4kod/api-network-monitor.git
cd api-network-monitor
```

2. Создайте виртуальное окружение и активируйте его:
```
python -m venv .venv
.venv\Scripts\activate  # для Windows
# или
source .venv/bin/activate  # для Linux/Mac
```

3. Установите зависимости:
```
pip install -r requirements.txt
```

## Запуск

Для запуска приложения выполните:
```
python start.py
```

После запуска приложение будет доступно по адресу: http://localhost:8000

## Структура проекта

- `main.py` - основной файл приложения с API-эндпоинтами
- `start.py` - скрипт для запуска приложения
- `templates/` - HTML-шаблоны для веб-интерфейса
- `static/` - статические файлы (CSS, JavaScript, изображения)
- `requirements.txt` - список зависимостей

## Использование

1. Откройте веб-интерфейс по адресу http://localhost:8000
2. Добавьте устройства для мониторинга, указав их IP-адрес, название и описание
3. Запустите мониторинг для нужных устройств
4. Отслеживайте статус устройств и историю изменений
